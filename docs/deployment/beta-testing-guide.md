## 베타 테스트 개요

### 목표
- **기능 검증**: 모든 핵심 기능 정상 작동 확인
- **버그 발견**: 실제 사용 환경에서 버그 및 문제점 발견
- **사용성 테스트**: UI/UX 개선점 파악
- **성능 검증**: 다양한 시스템 환경에서 성능 확인
- **피드백 수집**: 사용자 요구사항 및 개선 제안 수집

### 베타 테스트 기간
- **준비 기간**: 1일 (모집 및 배포)
- **테스트 기간**: 7일 (집중 테스트)
- **피드백 수집**: 테스트 기간 중 지속
- **결과 분석**: 1-2일

### 베타 테스트 범위
- **설치 및 첫 실행**
- **인증 시스템** (로그인, 회원가입, 기기 인증)
- **이미지 변환** (다양한 포맷 및 설정)
- **백업 및 복원**
- **로그 조회**
- **다국어 전환**
- **성능 및 안정성**

{
  "version": "0.1.0-beta.1"
}

# TypeScript 컴파일
npm run compile:main

# Webpack 빌드
npm run build

# Electron Builder 빌드
npm run build:win

## 베타 테스트 실행

### 1. 베타 테스트 킥오프

#### 킥오프 미팅 (선택 사항)
- **날짜**: 테스트 시작일
- **시간**: 30분
- **내용**:
  - 프로젝트 소개
  - 테스트 목표 설명
  - 테스트 항목 안내
  - 피드백 제공 방법 설명
  - Q&A

#### 테스트 가이드 배포
- **문서**: 베타 테스트 가이드 (이 문서)
- **체크리스트**: 테스트 항목 체크리스트
- **버그 리포트 템플릿**: GitHub Issues 템플릿

### 2. 테스트 항목 체크리스트

#### Phase 1: 설치 및 첫 실행 (Day 1)
- [ ] **설치 파일 다운로드**: GitHub Release에서 다운로드
- [ ] **설치 실행**: 더블 클릭으로 설치 시작
- [ ] **설치 경로 선택**: 기본 경로 또는 사용자 지정 경로
- [ ] **설치 완료**: 에러 없이 설치 완료
- [ ] **첫 실행**: 앱 정상 실행
- [ ] **UI 로딩**: 모든 UI 요소 정상 표시
- [ ] **언어 자동 감지**: 시스템 언어 자동 감지
- [ ] **언어 전환**: 한국어 ↔ 영어 전환 작동

**예상 소요 시간**: 10분

#### Phase 2: 인증 시스템 (Day 1-2)
- [ ] **회원가입**: 새 계정 생성
  - 이메일 입력
  - 비밀번호 입력
  - 비밀번호 확인
  - 회원가입 버튼 클릭
  - 성공 메시지 확인
- [ ] **로그인**: 생성한 계정으로 로그인
  - 이메일 입력
  - 비밀번호 입력
  - 로그인 버튼 클릭
  - 로그인 성공 확인
  - 사용자 정보 헤더 표시 확인
- [ ] **기기 인증**: 자동 기기 등록 확인
  - 기기 ID 자동 생성
  - registered_devices 테이블 등록 (Supabase 대시보드 확인)
- [ ] **로그아웃**: 로그아웃 버튼 작동 확인

**예상 소요 시간**: 15분

#### Phase 3: 이미지 변환 (Day 2-4)
- [ ] **파일 선택 (드래그 앤 드롭)**:
  - 이미지 파일 5개 준비 (JPG, PNG, GIF, TIFF, BMP)
  - 드롭존에 드래그 앤 드롭
  - 파일 목록 표시 확인
- [ ] **파일 선택 (다이얼로그)**:
  - "파일 선택" 버튼 클릭
  - 파일 선택 다이얼로그 열기
  - 다중 파일 선택
  - 파일 목록 추가 확인
- [ ] **변환 설정**:
  - **출력 포맷**: WebP, AVIF, JPG, PNG 선택
  - **품질**: 슬라이더로 0-100% 조절
  - **압축 레벨**: 슬라이더로 0-9 조절
  - **빠른 프리셋**: "빠른 변환", "균형잡힌", "최고 품질" 선택
- [ ] **출력 폴더 선택**:
  - "출력 폴더 선택" 버튼 클릭
  - 폴더 선택 다이얼로그 열기
  - 폴더 선택 확인
- [ ] **변환 실행**:
  - "변환 시작" 버튼 클릭
  - 진행 상태 실시간 표시 확인
  - 개별 파일 상태 표시 확인
  - 변환 완료 알림 확인
  - 출력 폴더에서 파일 생성 확인

**테스트 케이스**:
1. **WebP 변환 (품질 80%)**:
   - JPG 5개 → WebP 변환
   - 압축률 확인 (30-50%)
   - 처리 시간 확인 (< 3초/1MB)
2. **AVIF 변환 (품질 70%)**:
   - PNG 5개 → AVIF 변환
   - 압축률 확인 (40-60%)
   - 처리 시간 확인 (< 5초/1MB)
3. **JPG 변환 (품질 90%)**:
   - TIFF 2개 → JPG 변환
   - 품질 확인
4. **다중 포맷 변환**:
   - 혼합 파일 10개 → WebP 변환
   - 병렬 처리 확인 (4개 동시)

**예상 소요 시간**: 1-2시간

#### Phase 4: 백업 시스템 (Day 4-5)
- [ ] **자동 백업**:
  - 이미지 변환 시작
  - 백업 폴더 자동 생성 확인 (`backup/YYYY-MM-DD/`)
  - 백업 파일 생성 확인
  - 메타데이터 파일 생성 확인 (`.json`)
- [ ] **백업 조회**:
  - "백업" 탭 이동
  - 백업 목록 표시 확인
  - 통계 카드 정확성 확인 (총 백업, 활성, 복원, 사용 공간)
- [ ] **백업 복원**:
  - 백업 항목 선택
  - "복원" 버튼 클릭
  - 복원 확인 모달 표시
  - 복원 실행
  - 파일 복원 확인
  - 상태 업데이트 확인 ("restored")
- [ ] **백업 삭제**:
  - 백업 항목 선택
  - "삭제" 버튼 클릭
  - 삭제 확인 모달 표시
  - 삭제 실행
  - 백업 파일 삭제 확인
  - 목록에서 제거 확인

**예상 소요 시간**: 30분

#### Phase 5: 로그 시스템 (Day 5-6)
- [ ] **자동 로그 기록**:
  - 이미지 변환 완료 후 로그 자동 생성 확인
  - Excel 파일 생성 확인 (`logs/YYYY-MM-DD.xlsx`)
- [ ] **로그 조회**:
  - "로그" 탭 이동
  - 로그 목록 표시 확인
  - 통계 카드 정확성 확인 (총 파일, 성공률, 압축률, 절약 용량)
- [ ] **Excel 파일 확인**:
  - `logs/` 폴더 열기
  - Excel 파일 열기
  - "작업 기록" 시트 확인 (13개 컬럼)
  - "통계" 시트 확인
  - 하이퍼링크 작동 확인

**예상 소요 시간**: 20분

#### Phase 6: 다국어 및 기타 기능 (Day 6-7)
- [ ] **다국어 전환**:
  - 언어 전환 버튼 클릭
  - 한국어 → 영어 전환
  - 모든 UI 텍스트 번역 확인
  - 영어 → 한국어 전환
  - 설정 저장 확인 (재시작 후 유지)
- [ ] **구독 등급별 제한** (테스트 계정 필요):
  - Free 등급: WebP만 사용 가능, 배치 크기 5개
  - Basic 등급: WebP, AVIF 사용 가능, 배치 크기 15개
  - Pro 등급: 모든 포맷 사용 가능, 무제한 배치
- [ ] **어필리에이트 시스템** (선택 사항):
  - "어필리에이트" 탭 이동
  - 추적 링크 생성
  - 통계 조회

**예상 소요 시간**: 30분

#### Phase 7: 성능 및 안정성 (Day 7)
- [ ] **메모리 사용**:
  - Task Manager 열기
  - Idle 시 메모리 확인 (< 500MB 목표)
  - 변환 중 메모리 확인 (< 1GB 목표)
- [ ] **CPU 사용**:
  - Task Manager 열기
  - Idle 시 CPU 확인 (< 5% 목표)
  - 변환 중 CPU 확인 (50-70% 목표)
- [ ] **안정성 테스트**:
  - 10개 파일 × 10회 반복 변환
  - 앱 크래시 없음 확인
  - 메모리 누수 없음 확인
- [ ] **대용량 파일 테스트**:
  - 10MB 이상 이미지 변환
  - 처리 시간 확인 (< 20초)
  - 에러 없음 확인

**예상 소요 시간**: 1시간

### 3. 자유 사용 테스트 (Day 3-7 병행)

베타 테스터는 구조화된 테스트 외에도 자유롭게 앱을 사용하며 다음을 테스트합니다:
- **일상 워크플로우**: 실제 작업 환경에서 사용
- **예외 케이스**: 비정상적인 사용 패턴 시도
- **엣지 케이스**: 극단적인 상황 테스트
- **UI/UX 평가**: 직관성, 편의성, 디자인 평가

## 결과 분석

### 1. 버그 분류 및 우선순위

#### Critical (즉시 수정)
- 앱 크래시
- 데이터 손실
- 보안 취약점

#### High (빠른 수정)
- 핵심 기능 작동 불가
- 로그인 불가
- 이미지 변환 실패

#### Medium (다음 버전)
- 일부 기능 작동 불가
- 백업 실패
- 로그 미생성

#### Low (장기 계획)
- UI 버그
- 텍스트 오타
- 정렬 문제

### 2. 피드백 통계 분석

```markdown
## 베타 테스트 결과 보고서

### 참여 현황
- 총 참여 인원: 15명
- 완료율: 80% (12명)
- 평균 테스트 시간: 3시간

### 버그 리포트
- 총 버그 수: 25개
  - Critical: 2개 (8%)
  - High: 5개 (20%)
  - Medium: 10개 (40%)
  - Low: 8개 (32%)

### 기능 제안
- 총 제안 수: 15개
  - High: 3개 (20%)
  - Medium: 8개 (53%)
  - Low: 4개 (27%)

### 사용성 평가 (평균 점수)
- 설치 및 첫 실행: 4.5/5
- 인증 시스템: 4.2/5
- 이미지 변환: 4.8/5
- 백업 및 로그: 4.0/5
- 전반적인 만족도: 4.4/5

### 추천 의향
- 추천 의향: 85% (예)
- 정식 버전 사용 의향: 90% (예)

### 주요 긍정 피드백
1. "이미지 변환 속도가 빠르고 압축률이 좋습니다."
2. "UI가 직관적이고 사용하기 쉽습니다."
3. "백업 기능이 유용합니다."

### 주요 개선 제안
1. "더 많은 포맷 지원 필요 (예: HEIF, HEIC)"
2. "배치 처리 크기 제한 완화"
3. "프로그레스 바 더 자세히 표시"
```

### 3. 다음 단계 결정

#### Critical/High 버그 수정 (v0.1.1 패치)
- 즉시 수정 및 재배포
- 베타 테스터에게 업데이트 공지

#### Medium 버그 수정 (v0.2.0)
- 다음 메이저 버전에 포함
- 2-4주 내 수정 계획

#### Low 버그 수정 (v0.3.0)
- 장기 계획
- 우선순위 낮음

#### 기능 제안 반영 (v0.2.0, v0.3.0)
- High 우선순위 기능: v0.2.0에 포함
- Medium/Low 기능: v0.3.0 이후

## 베타 테스터 혜택

### 1. Pro 플랜 1개월 무료
- 베타 테스트 완료 후 제공
- 모든 Pro 기능 무제한 사용
- 자동 업데이트 우선 제공

### 2. 정식 버전 Pro 플랜 50% 할인
- 정식 버전 출시 시 50% 할인 쿠폰
- 1년 구독 기준
- 평생 할인 (갱신 시마다)

### 3. 베타 테스터 배지
- 앱 내 베타 테스터 배지 표시
- 프로필 페이지에 배지 표시
- 우선 지원 혜택

### 4. 개발 로드맵 우선 공개
- 다음 버전 기능 미리 확인
- 기능 제안 우선 반영
- 새로운 베타 테스트 우선 참여
