# 프로젝트: 더카당 픽셀부스터 (The Kadang Pixel Booster) 기획서

---

## 1. 프로젝트 개요

* **프로젝트명:** 더카당 픽셀부스터 (The Kadang Pixel Booster)
* **프로젝트 목표:** 윈도우(Windows) 및 맥(Mac) 환경에서 사용자가 직관적이고 현대적인 UI를 통해 이미지 파일을 손쉽게 변환/최적화할 수 있는 다국어 지원 데스크톱 애플리케이션을 개발하고, 기능 기반 구독(SaaS) 모델을 통해 글로벌 수익을 창출합니다.
* **핵심 가치:**
    * **작업 효율:** 강력한 일괄 처리(Batch processing) 기능과 상세한 로그 기록으로 반복 작업을 자동화합니다.
    * **성능 향상:** 이미지를 고압축/고효율 형식(WEBP, AVIF 등)으로 변환하여 웹사이트 로딩 속도를 개선합니다.
    * **직관적 경험:** 사용자가 설명서 없이도 즉시 사용할 수 있는 현대적이고 직관적인 UI/UX를 제공합니다.
    * **글로벌 접근성:** 다국어 지원을 통해 전 세계 사용자가 언어 장벽 없이 서비스를 이용할 수 있도록 합니다.
    * **수익 모델:** 매력적인 무료 버전을 기반으로 한 기능별 프리미엄(Freemium) 구독 모델과 어필리에이트 시스템을 결합합니다.

## 2. 핵심 기능 (애플리케이션)

### 2.1. UI/UX 디자인 원칙

* **직관성 (Intuitive):** 모든 기능은 사용자가 고민 없이 즉시 사용할 수 있도록 명확하게 배치됩니다.
* **현대성 (Modern):** 깔끔한 타이포그래피, 최소한의 색상 사용, 부드러운 인터랙션을 포함하는 최신 디자인 트렌드를 반영합니다.
* **시각적 피드백:** 파일 드래그 앤 드롭 영역, 변환 진행률 표시줄(Progress bar), 완료/오류 메시지를 명확하게 시각화하여 사용자에게 현재 상태를 즉시 알립니다.
* **최소 클릭:** 사용자가 원하는 작업을 완료하기까지 거치는 단계를 최소화하도록 동선을 설계합니다.

### 2.2. 다국어 지원 (Internationalization, i18n)

* 애플리케이션 실행 시 사용자의 OS(운영체제) 시스템 언어를 기본으로 자동 설정합니다.
* 사용자가 앱 내 설정에서 언제든지 언어를 수동으로 변경할 수 있는 선택 메뉴를 제공합니다.
* **초기 지원 언어:** 한국어, 영어 (향후 JSON 파일 추가만으로 다른 언어 확장이 용이하도록 설계합니다.)

### 2.3. 변환 설정

* **변환 형식 선택 (구독별 차등):**
    * **Free:** **WEBP** 형식으로만 변환 가능
        * 입력: JPG, PNG, GIF, BMP, TIFF, SVG → 출력: WEBP
    * **Basic:** **WEBP, AVIF** 형식으로 변환 가능
        * 입력: JPG, PNG, GIF, BMP, TIFF, SVG, HEIF, AVIF → 출력: WEBP, AVIF
    * **Pro:** **모든 주요 이미지 포맷 상호 변환** 지원
        * 입력: JPG, PNG, GIF, BMP, TIFF, SVG, WEBP, AVIF, HEIF
        * 출력: JPG, PNG, GIF, WEBP, AVIF, TIFF
        * 예시: `PNG ↔ JPG`, `GIF → WEBP`, `HEIF → AVIF` 등 자유로운 변환
* **작동 범위 선택 (구독별 차등):**
    * **Free:** **파일 선택** (다중 선택 가능)
    * **Basic:** **폴더 선택** (다중 선택 및 하위 폴더 포함 옵션 지원)
    * **Pro:** **컴퓨터 전체** (지정된 드라이브 또는 전체 시스템을 스캔하여 이미지 변환)

### 2.4. 작동 방식 (구독별 차등)

* **파일명 정책:** 변환된 파일의 이름은 원본 파일명과 동일하게 유지되며, 확장자만 변경됩니다.
* **방식 선택:**
    * **Free:** **이미지 변환** (원본 파일을 백업 없이 덮어쓰거나 지정된 새 폴더에 저장)
    * **Basic 이상:** **변환 및 백업** 기능 추가 (원본 이미지를 변환하며, 원본 파일은 해당 폴더 내 'backup' 폴더를 자동 생성하여 이동)

### 2.5. 로그 기록 (Basic 등급 이상)

* **파일 형식:** 모든 작업 내역은 **Excel (.xlsx)** 파일로 관리됩니다.
* **로그 위치:** 작동 시(폴더/파일 선택 기준) 해당 폴더 내에 로그 파일을 자동 생성 또는 업데이트합니다.
* **시트 관리:** 작업이 실행된 '작동 일시' (예: `2025-11-10 13:30`)를 기준으로 새 시트(Sheet)를 생성하고 작업 내역을 기록합니다.
* **기록 내용:** 작동 범위, 작동 방식, 원본 파일명, 적용된 파일명 (확장자 포함)
* **하이퍼링크:** '적용된 파일명' 셀을 클릭하면 해당 변환 완료된 파일이 열리도록 링크를 처리합니다.

## 3. 비즈니스 모델 및 서버 연동

### 3.1. 구독 모델 (Freemium)

* **서버 연동:** 사용자의 구독 상태(등급, 만료일)는 서버에서 인증 및 관리합니다. 애플리케이션 실행 시 서버와 통신하여 구독 상태를 확인하고, 이에 따라 사용 가능한 기능을 활성화/비활성화합니다.
* **변환 횟수:** **모든 등급(Free 포함)에서 변환 횟수 제한은 없습니다.**
* **등급별 기능 제한 (요약 테이블):**

| 기능 | Free (무료) | Basic (베이직) | Pro (프로) |
| :--- | :--- | :--- | :--- |
| **변환 횟수** | **무제한** | **무제한** | **무제한** |
| **변환 형식** | **WEBP** 로만 변환<br>(입력: JPG, PNG, GIF, BMP, TIFF, SVG) | **WEBP, AVIF**<br>(입력: Free + HEIF, AVIF) | **모든 포맷 상호 변환**<br>(입력: Basic + WEBP)<br>(출력: JPG, PNG, GIF, WEBP, AVIF, TIFF) |
| **작동 범위** | 파일 선택 | **+ 폴더 선택** | **+ 컴퓨터 전체** |
| **작동 방식** | 기본 변환 | **+ 변환 및 백업** | + 변환 및 백업 |
| **로그 기록** | ❌ | ✅ | ✅ |
| **다국어 지원** | ✅ | ✅ | ✅ |

### 3.2. 관리자 대시보드 (Web)

* 관리자(운영자)가 사용자 및 수익을 관리할 수 있는 웹 기반 대시보드를 구축합니다.
* **주요 기능:**
    * **사용자 관리:** 전체 사용자 목록, 아이디별 구독 현황(Free, Basic, Pro) 확인.
    * **수익 관리:** 일별/월별/사용자별 수익 현황 및 통계 확인.
    * **권한 설정:** 관리자가 특정 아이디의 구독 권한을 수동으로 부여, 변경, 회수할 수 있습니다.

### 3.3. 어필리에이트 시스템

* **목표:** 초기 사용자 확보 및 마케팅 채널 다각화
* **작동 방식:**
    1.  파트너(어필리에이터)에게 고유 트래킹 링크를 발급합니다.
    2.  사용자가 해당 링크를 클릭 시 **3일간** 쿠키(Cookie)가 저장됩니다.
    3.  사용자가 3일 이내 구독 상품(Basic, Pro)을 결제하면 해당 파트너의 추천으로 기록됩니다.
* **수익 배분:**
    * **지급 조건:** 추천받은 이용자가 **구독을 유지하는 내내** (구독 갱신 시마다) 수익의 일부를 파트너에게 지급합니다.
    * **기본 수수료:** 30%
    * **수수료 관리:** 관리자 대시보드에서 파트너 아이디별로 수수료 퍼센트(%)를 개별 설정할 수 있습니다.

## 4. 기술 사양 및 개발

* **데스크톱 앱 (클라이언트):**
    * **플랫폼:** Windows, macOS
    * **권장 기술:** **Electron** 또는 **Tauri**. (크로스 플랫폼 개발 및 웹 기술(JS/TS) 활용에 용이)
* **서버 (백엔드):**
    * **역할:** 사용자 인증, 구독 관리(결제 연동), 어필리에이트 로직 처리
    * **권장 기술:** Node.js(Express), PHP(Laravel), Python(Django) 등
    * **데이터베이스:** MySQL 또는 PostgreSQL
* **다국어 구현:**
    * `i18next`와 같은 국제화 라이브러리를 사용하여 언어 리소스(JSON 파일)를 관리하고 쉽게 확장할 수 있도록 구축합니다.
* **자동 업데이트 시스템:**
    * **GitHub Releases**와 `electron-updater` (Electron 사용 시) 또는 Tauri의 내장 업데이터를 연동하여, 새 버전 배포 시 사용자에게 알리고 자동으로 다운로드 및 설치를 진행하도록 구현합니다.
